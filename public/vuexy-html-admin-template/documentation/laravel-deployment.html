Laravel Deployment

We are going to discuss basic changes in our template before deploying it.

Note: Make sure that you have successfully run our template locally before deploying it. Follow our installation docs for more details.

Introduction

Laravel deployment differs from server to server and also if you deploy on root domain or subdomains. There are no special steps of deployment specifically for Vuexy, it's the same for all the projects built with Laravel.

Note: Here, we discuss, the basic changes you have to do before deploying Vuexy template. For laravel deployment tips and tricks follow its official documentation: https://laravel.com/docs/11.x/deployment

Root Folder Deployment

There should be some minor changes when you deploy your project on root folder or sub folder.

  1. First you need to change your .env variables value:
    2. APP_ENV=production
APP_DEBUG=false
  2. Now, run the following command to generate the production build for deployment.
    3. yarn build
  3. Our template is ready to deploy. Here, we assume a simple folder structure to deploy the laravel app on the server. It may differ from server to server..

    4. We have two folders on the server.

    • public_html
    • laravel

    We move all public files inside the public_html folder, and the rest of our application resides in the laravel folder.

  1. Follow step 1 and step 2 of the above root deployment steps.
